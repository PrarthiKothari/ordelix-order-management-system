# Ordelix - Order Management System (SQL Project)

## Overview
Ordelix, an Order Management System (OMS) is a structured database-driven system designed to manage and monitor the complete lifecycle of customer orders in an e-commerce environment. This project is built entirely using SQL with 
MariaDB on XAMPP, focusing specifically on the backend data layer, business rules, and operational workflows of a real-world e-commerce platform. 

## Purpose
To design and implement a professional, scalable, and efficient SQL-based Order Management System that reflects real-world industry practices.

This project aims to demonstrate: 
1. Strong understanding of database design 
2. Practical implementation of relational modeling and performance optimization 
3. Usage of constraints, foreign keys, normalization, automation, stored procedures, views 
4. Implementation of business logic through SQL 
5. Support for inventory synchronization and order lifecycle management 
6. Optimization for analytics and reporting 

## Features
- Customer Registration and Profile Storage
- Address Management
- Product Catalog Organization
- Warehouse and Inventory Tracking
- Cart Handling
- Order Placement
- Payment Recording
- Shipment Management
- System Activity Logging 

## Technologies Used
- MySQL (MariaDB)
- SQL (Joins, Views, Stored Procedures, Triggers)
